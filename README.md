# nim-install

A CLI that generates AI agent model configurations for
[NVIDIA NIM](https://build.nvidia.com/) — the enterprise-grade hosted inference
service for the world's leading open models.

It fetches the live catalog of available models directly from the NVIDIA NIM
API, so configs are always up to date as models are added or retired.

**Supported targets**

| Target | Status | Config location |
|---|---|---|
| [Continue](https://continue.dev) | ✅ Available | `~/.continue/config.yaml` |
| pi-agent | 🔜 Planned | — |
| GitHub Copilot | 🔜 Planned | — |

---

## Prerequisites

- **Node.js ≥ 18** (for native `fetch`)
- An **NVIDIA NIM API key** — get one free at
  [build.nvidia.com](https://build.nvidia.com/)

## Installation

Install globally straight from GitHub:

```bash
npm install -g github:Ober3550/nim-install
```

To pin to a specific version or tag:

```bash
npm install -g github:Ober3550/nim-install#v1.0.0
```

The compiled output is committed to the repository, so no build step is
required at install time.

### Upgrading

```bash
npm install -g github:Ober3550/nim-install
```

### Uninstalling

```bash
npm uninstall -g nim-install
```

## Usage

### Interactive mode

Running the command without arguments launches an interactive wizard:

```bash
nim-install generate
```

You will be prompted to:

1. Choose a target agent (e.g. Continue)
2. Enter your NVIDIA NIM API key (or export `NVIDIA_API_KEY` first)
3. Select which models to include (all chat / code models are pre-selected)
4. Confirm the output path

### Non-interactive / scripted mode

```bash
nim-install generate continue \
  --api-key nvapi-xxxxxxxxxxxx \
  --output ~/.continue/config.yaml \
  --all-models \
  --overwrite
```

| Flag | Default | Description |
|---|---|---|
| `[target]` | interactive | Target agent: `continue` (more coming) |
| `-k, --api-key <key>` | `$NVIDIA_API_KEY` | Your NVIDIA NIM API key |
| `-o, --output <path>` | agent default | Where to write the config |
| `--all-models` | off | Skip model selection, include all chat/code models |
| `--overwrite` | off | Overwrite an existing config file |

### Environment variable

Set your key once so you never have to pass it on the command line:

```bash
export NVIDIA_API_KEY="nvapi-xxxxxxxxxxxx"
nim-install generate continue --all-models --overwrite
```

### Listing available models

```bash
# Human-readable with model type annotations
nim-install list-models

# Raw JSON (useful for scripting)
nim-install list-models --json
```

## What gets generated

### Continue (`~/.continue/config.yaml`)

The generator creates a `config.yaml` with one entry per selected model,
using Continue's OpenAI-compatible provider pointed at the NIM API:

```yaml
# Generated by nim-install
# SECURITY NOTE: This file contains your NVIDIA NIM API key.
# Keep this file private and do not commit it to version control.

name: NVIDIA NIM
version: 1.0.0
schema: v1
models:
  - name: Meta — Llama 3.1 70B Instruct
    provider: openai
    model: meta/llama-3.1-70b-instruct
    apiBase: https://integrate.api.nvidia.com/v1
    requestOptions:
      headers:
        Authorization: Bearer nvapi-xxxxxxxxxxxx
    capabilities:
      - tool_use
    roles:
      - chat
      - edit
      - apply

  - name: Qwen — Qwen2.5 Coder 7B Instruct
    provider: openai
    model: qwen/qwen2.5-coder-7b-instruct
    apiBase: https://integrate.api.nvidia.com/v1
    requestOptions:
      headers:
        Authorization: Bearer nvapi-xxxxxxxxxxxx
    capabilities:
      - tool_use
    roles:
      - chat
      - edit
      - apply

  - name: Qwen — Qwen2.5 Coder 7B Instruct (Autocomplete)
    provider: openai
    model: qwen/qwen2.5-coder-7b-instruct
    apiBase: https://integrate.api.nvidia.com/v1
    requestOptions:
      headers:
        Authorization: Bearer nvapi-xxxxxxxxxxxx
    capabilities: []
    roles:
      - autocomplete
```

Code models (those identified as supporting Fill-in-Middle) are added with
both a `chat/edit/apply` entry and a separate `autocomplete` entry.

Specialist models (embeddings, rerankers, image generation, scientific /
healthcare models, safety classifiers) are automatically excluded from the
chat config.

## Security

> **Your API key is stored in plain text in the generated config file.**

Best practices:

- Keep `~/.continue/config.yaml` out of version control (add it to
  `~/.gitignore_global` or your project's `.gitignore`).
- Rotate your key at [build.nvidia.com](https://build.nvidia.com/) if it is
  ever exposed, then re-run `nim-install generate` to update your configs.
- Never share the config file with anyone.

## Adding support for a new agent

1. Create `src/generators/<agent-name>.ts` and implement the `Generator`
   interface from `src/generators/types.ts`:

   ```typescript
   import type { Generator, GeneratorOptions } from './types';

   export const myAgentGenerator: Generator = {
     id: 'my-agent',
     name: 'My Agent',
     description: 'Short description shown in the CLI',
     defaultOutputPath: '/path/to/default/config',
     async generate(options: GeneratorOptions): Promise<void> {
       // Write the config file for your agent using options.models,
       // options.apiKey, and options.apiBase
     },
   };
   ```

2. Register it in `src/generators/registry.ts`:

   ```typescript
   import { myAgentGenerator } from './my-agent';

   const generators = new Map<string, Generator>([
     ['continue', continueGenerator],
     ['my-agent', myAgentGenerator],
   ]);
   ```

3. Rebuild: `npm run build`

4. The new target is now available: `nim-install generate my-agent`
